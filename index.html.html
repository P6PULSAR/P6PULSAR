Schedule Quality Dashboard — computed from Current XER only
Why two scores? The DCMA-14 Health Score above and the SCQ Score below are built from different checklists on purpose — they're two different rulers, not two attempts at the same number, and they won't always agree. DCMA-14 blends network-structure checks (logic, constraints, float, duration) with execution-performance checks — CPLI and BEI — that reflect whether the project is actually running to plan. SCQ Score looks only at network/logic quality: circular logic, redundant ties, logic on summary activities, out-of-sequence and reverse logic, merge hotspots, and logic density — none of which DCMA-14 measures. A schedule can have clean logic (a good SCQ score) while running behind plan (a poor DCMA score), or the reverse — read them side by side, not against each other.

Assumed definitions & thresholds — Hard Constraints: Must Start On / Must Finish On / Mandatory Start / Mandatory Finish. Soft Constraints: Start On-or-Before / Start On-or-After / Finish On-or-Before / Finish On-or-After (informational only, not scored). Insufficient Detail: duration > days (same limit as DCMA #8 — editable in Threshold Settings above). Merge Hotspot: more than predecessors (editable in Threshold Settings above). Redundancy Index: a direct logic tie that is also reachable via another path through the network — flagged when it affects more than 10% of relationships (fixed this release, not user-editable). Out of Sequence: a successor shows actual progress before its predecessor relationship logic is satisfied — flagged above 5% of relationships (fixed). Circular Logic, Logic on Summaries, and Reverse Logic are zero-tolerance — any single occurrence fails. Logic Density = relationships ÷ activities: ≥2.0 good, 1.0–2.0 marginal, <1.0 poor. The Redundancy Index scan is skipped on schedules with more than 15,000 relationships as a performance safeguard.
